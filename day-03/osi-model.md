# OSI Model — Mail Analogy

1. **Physical** – the truck and roads carrying the mail. Actual wires/signals moving raw bits.
2. **Data Link** – the local post office sorting by street. MAC addresses move frames between devices on the same network.
3. **Network** – the postal routing system between cities. IP addresses route packets across networks.
4. **Transport** – tracking number + delivery confirmation. TCP/UDP ensures (or doesn't) reliable delivery of data.
5. **Session** – the phone call setup before a conversation. Manages opening/maintaining/closing the connection.
6. **Presentation** – translating the letter's language. Handles encryption/formatting so both sides understand the data.
7. **Application** – the actual letter's content. What the user interacts with — browser, email client, etc.
